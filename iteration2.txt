Customer needs: 
 
I need to talk to an LLM and get answers to my questions. I would like to keep my conversation 
history so I can view and search it. I also want to continue my previous conversations. 
 
New: for new features, the development team works in a new branch, then submits pull 
requests (PRs) once the features are tested. The testing team needs to review the PR and 
ensure the new features pass the required tests before merging. The requirement team needs 
to keep track of the mapping between features and PRs in a change log.  
 
Change log format: create a text file in the repository with the name of change.log. 
Sample change log: 
Feature 4: Save conversation history. [PR 3](link to PR3)  
Feature 5: Search conversation history. [PR4](link to PR4)
